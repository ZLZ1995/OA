# 资产评估项目流程管理系统 - 架构固化（阶段1）

## 1. 技术架构
- 前端：Vue3 + Vite + Element Plus（阶段3实施）
- 后端：FastAPI + SQLAlchemy + Pydantic
- 数据库：PostgreSQL
- 文件存储：MinIO（后续通过对象存储抽象兼容云厂商）
- 鉴权：JWT
- 权限：固定角色模板 + 多角色绑定 + 后端强校验
- 流程：轻量状态机（非重型 BPM）

## 2. 目录约定
- `backend/app/models`：数据模型
- `backend/app/workflows`：状态机、迁移规则、守卫规则
- `backend/app/services`：服务层（业务编排与规则校验）
- `backend/app/api/v1`：对外接口
- `docs/*.md`：阶段1固化文档

## 3. 核心业务流程
项目创建 → 工单创建 → 合同上传 → 文印室正式合同出具 → 项目负责人提交一审并选一审老师 →
一审老师审核 → 一审通过返回项目负责人 → 项目负责人提交二审并选二审老师 →
二审老师审核 → 二审通过返回项目负责人 → 项目负责人提交三审并选三审老师 →
三审老师审核并通过转文印室 → 文印室出具纸质报告。

## 4. 非功能约束
- 所有敏感接口必须登录鉴权。
- 权限以后端判定为准，前端仅负责体验层隐藏。
- 所有流程动作必须写入 `workflow_logs`。
